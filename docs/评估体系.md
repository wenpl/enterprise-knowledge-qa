## 一、评估指标体系

### 1.1 核心指标

| 指标 | 定义 | 计算方式 | 目标值 |
|------|------|----------|--------|
| **检索准确率** | 返回的文档是否与问题相关 | 相关文档数 / 返回文档数 | ≥ 85% |
| **答案相关性** | 答案是否回答了用户的问题 | 人工评分1-5分 | ≥ 4.0 |
| **答案准确性** | 答案内容是否正确 | 人工评分1-5分 | ≥ 4.0 |
| **引用准确性** | 引用的来源是否正确 | 正确引用数 / 总引用数 | ≥ 90% |

### 1.2 过程指标

| 指标 | 定义 | 数据来源 |
|------|------|----------|
| 平均响应时间 | 从提问到返回答案的时间 | 系统日志 |
| 用户满意度 | 用户对答案的评分 | 用户反馈 |
| Badcase率 | 效果不佳的问答占比 | 人工评估 |

## 二、评估方法

### 2.1 自动评估

适用于：检索准确率

```python
def evaluate_retrieval(query, retrieved_docs, ground_truth):
    """
    评估检索效果

    Args:
        query: 查询问题
        retrieved_docs: 检索返回的文档列表
        ground_truth: 标注的正确文档

    Returns:
        precision: 准确率
    """
    relevant_count = 0
    for doc in retrieved_docs:
        if doc['source'] in ground_truth:
            relevant_count += 1

    precision = relevant_count / len(retrieved_docs)
    return precision
```

### 2.2 人工评估

适用于：答案相关性、答案准确性

#### 评估流程

```
1. 准备测试问题集
2. 系统生成答案
3. 评估人员打分
4. 统计分析结果
```

#### 评分标准

**答案相关性评分标准**

| 分数 | 描述 |
|------|------|
| 5分 | 完全回答了问题，内容完整 |
| 4分 | 基本回答了问题，有小遗漏 |
| 3分 | 部分回答了问题 |
| 2分 | 答案与问题关联较弱 |
| 1分 | 答案与问题无关 |

**答案准确性评分标准**

| 分数 | 描述 |
|------|------|
| 5分 | 答案内容完全正确 |
| 4分 | 答案内容基本正确，有小错误 |
| 3分 | 答案内容部分正确 |
| 2分 | 答案内容大部分错误 |
| 1分 | 答案内容完全错误 |

## 五、Badcase分析

### 5.1 Badcase分类

| 类型 | 描述 | 可能原因 | 解决方案 |
|------|------|----------|----------|
| **检索失败** | 没有找到相关文档 | 文档未覆盖、切分不当 | 补充文档、优化切分 |
| **答案错误** | 答案内容错误 | Prompt问题、模型幻觉 | 优化Prompt |
| **答案不完整** | 遗漏关键信息 | 检索不全、生成不全 | 增加检索数量、优化Prompt |
| **格式问题** | 输出格式不规范 | Prompt格式要求不明确 | 强化格式要求 |

### 5.2 Badcase记录模板

```markdown
## Badcase #001

**问题**: 年假怎么算？

**期望答案**: 根据工龄计算，满1年5天，满5年10天，满10年15天

**实际答案**: 年假有5天

**问题类型**: 答案不完整

**原因分析**: 检索只返回了一个片段，缺少完整信息

**解决方案**: 增加检索数量，优化Prompt要求完整回答

**状态**: 已修复 ✅
```
