## 一、Prompt设计原则

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **准确性** | 只基于提供的文档内容回答，不编造信息 |
| **简洁性** | 回答简洁明了，直接回答问题 |
| **可追溯** | 标注信息来源，便于验证 |
| **边界清晰** | 明确告知无法回答的情况 |

### 1.2 设计考虑

- 用户可能是非技术人员，避免过于技术化的表达
- 答案需要可验证，标注来源很重要
- 需要处理文档中没有相关信息的情况

## 二、System Prompt设计

### 版本1.0（当前版本）

```
你是一个企业知识库问答助手。你的任务是基于提供的文档内容回答用户问题。

## 回答规范

1. 只基于提供的文档内容回答，不要编造信息
2. 如果文档中没有相关信息，请明确告知用户"文档中未找到相关信息"
3. 回答要简洁明了，直接回答问题
4. 如果答案来自多个文档片段，请综合整理
5. 在回答末尾标注信息来源

## 回答格式

【答案】
[你的答案内容]

【来源】
- [文档名称]

## 限制条件

- 不要回答与文档无关的问题
- 不要编造文档中不存在的内容
```

### 设计要点说明

| 部分 | 作用 | 设计理由 |
|------|------|----------|
| 角色定义 | 明确AI的身份和任务 | 建立正确的回答预期 |
| 回答规范 | 规定回答的具体要求 | 确保答案质量和一致性 |
| 回答格式 | 规定输出的格式 | 便于用户阅读和验证 |
| 限制条件 | 明确边界 | 避免不当回答 |

## 三、User Prompt模板

### 模板结构

```
## 文档内容
{context}

## 用户问题
{question}

请基于以上文档内容回答用户问题。如果文档中没有相关信息，请明确告知。
```

### 变量说明

| 变量 | 来源 | 说明 |
|------|------|------|
| {context} | RAG检索结果 | 相关文档片段的拼接 |
| {question} | 用户输入 | 用户的问题 |

## 五、Few-shot示例（可选优化）

### 示例设计

```
## 示例1

文档内容：
员工手册规定：年假天数根据工龄确定。入职满1年享有5天年假，入职满5年享有10天年假，入职满10年享有15天年假。

用户问题：年假有几天？

【答案】
年假天数根据工龄确定：
- 入职满1年：5天
- 入职满5年：10天
- 入职满10年：15天

【来源】
- 员工手册.md

## 示例2

文档内容：
API接口文档说明：登录接口地址为 POST /api/v1/auth/login，需要参数username和password。

用户问题：怎么登录系统？

【答案】
根据API文档，登录需要调用登录接口：
- 接口地址：POST /api/v1/auth/login
- 参数：username（用户名）、password（密码）

【来源】
- API接口文档.md
```

## 六、优化策略

### 6.1 效果评估维度

| 维度 | 评估方法 | 优化方向 |
|------|----------|----------|
| 准确性 | 人工对比文档 | 调整Prompt约束 |
| 完整性 | 检查是否遗漏关键信息 | 增加完整性提示 |
| 简洁性 | 检查是否冗余 | 添加简洁性要求 |
| 格式规范 | 检查输出格式 | 强化格式要求 |

### 6.2 常见问题与解决

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 答案编造 | Prompt约束不够 | 强化"只基于文档"要求 |
| 答案过长 | 没有长度限制 | 添加"简洁回答"要求 |
| 格式不规范 | 格式要求不明确 | 提供格式示例 |
| 来源标注缺失 | 没有强调来源 | 强化来源标注要求 |

## 七、成本优化

### Token消耗分析

| 部分 | Token数 | 占比 |
|------|---------|------|
| System Prompt | ~150 | 固定 |
| 文档内容 | ~500-2000 | 变动 |
| 用户问题 | ~20-50 | 变动 |
| 答案生成 | ~100-300 | 变动 |

### 优化建议

1. **控制检索文档数量**：TOP_K=3，避免过多无关内容
2. **精简System Prompt**：去除冗余描述
3. **限制答案长度**：设置max_tokens
