## 二、核心模块设计

### 2.1 文档处理模块

#### 功能描述
将上传的文档进行处理，转换为可检索的向量表示。

#### 处理流程

```
原始文档 → 文档加载 → 文档切分 → 向量化 → 存储
```

#### 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| chunk_size | 500 | 每个文档片段的字符数 |
| chunk_overlap | 50 | 相邻片段的重叠字符数 |

#### 代码示例

```python
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_community.document_loaders import TextLoader

def process_document(file_path):
    # 1. 加载文档
    loader = TextLoader(file_path)
    documents = loader.load()

    # 2. 切分文档
    text_splitter = RecursiveCharacterTextSplitter(
        chunk_size=500,
        chunk_overlap=50
    )
    chunks = text_splitter.split_documents(documents)

    return chunks
```

### 2.2 向量检索模块

#### 功能描述
将用户问题转换为向量，在向量数据库中检索相关文档片段。

#### 检索流程

```
用户问题 → 向量化 → 相似度计算 → 返回Top-K文档
```

#### 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| top_k | 3 | 返回最相关的K个文档片段 |
| score_threshold | 0.7 | 相似度阈值 |

### 2.3 答案生成模块

#### 功能描述
基于检索到的文档片段，使用LLM生成答案。

#### 生成流程

```
问题 + 相关文档 → Prompt组装 → LLM调用 → 答案生成
```

## 四、Prompt设计

### 4.1 System Prompt

```
你是一个企业知识库问答助手。你的任务是基于提供的文档内容回答用户问题。

## 回答规范

1. 只基于提供的文档内容回答，不要编造信息
2. 如果文档中没有相关信息，请明确告知用户
3. 回答要简洁明了，直接回答问题
4. 如果答案来自多个文档片段，请综合整理
5. 在回答末尾标注信息来源

## 回答格式

【答案】
[你的答案内容]

【来源】
- [文档名称]: [相关内容摘要]

## 限制条件

- 不要回答与文档无关的问题
- 不要泄露敏感信息
- 不要编造文档中不存在的内容
```

### 4.2 用户Prompt模板

```
## 文档内容
{context}

## 用户问题
{question}

请基于以上文档内容回答用户问题。
```

### 7.2 优化建议

1. **缓存热门问题的答案**：减少重复调用
2. **优化Prompt长度**：减少token消耗
